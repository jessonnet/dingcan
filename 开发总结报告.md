### 项目信息

| 项目名称 | 饭堂订餐系统 |
|---------|------------|
| 项目类型 | Web应用系统 |
| 开发语言 | Java + JavaScript |
| 当前版本 | V1.2.3 |
| 最后更新 | 2026-02-06 |
| 开发者 | 张志松 |

### 项目简介

饭堂订餐系统是一个面向企业内部员工的在线订餐管理系统，支持多种登录方式，提供便捷的订餐服务。系统采用前后端分离架构，后端使用Spring Boot框架，前端使用Vue.js框架，数据库采用MySQL。

### 核心特性

- ✅ 双登录模式：支持账号密码登录和微信登录
- ✅ 微信登录绑定：首次使用微信登录时绑定员工账号
- ✅ 鸿蒙系统适配：自动检测鸿蒙设备，优化显示效果
- ✅ 角色权限管理：支持员工、厨师、管理员三种角色
- ✅ 订餐管理：员工可以方便地预订餐食
- ✅ 订单管理：厨师可以查看和管理订单
- ✅ 系统配置：管理员可以配置系统参数
- ✅ 操作日志：记录所有操作，便于审计

## 功能模块

### 1. 用户认证模块

#### 1.1 账号密码登录

**功能描述**：
- 支持员工使用账号密码登录
- 账号锁定机制：连续5次失败后锁定30分钟
- 密码强度验证：至少8位，包含字母和数字
- JWT Token认证：登录成功后返回Token

**主要接口**：
- `POST /api/auth/login` - 登录接口
- `POST /api/auth/logout` - 登出接口

**安全特性**：
- 密码加密存储（BCrypt）
- 账号锁定防止暴力破解
- Token过期时间24小时

#### 1.2 微信登录

**功能描述**：
- 支持微信一键登录
- 首次登录需要绑定员工账号
- 已绑定用户直接登录
- 自动检测微信环境

**主要接口**：
- `GET /api/wechat/auth/url` - 获取微信授权URL
- `POST /api/wechat/login` - 微信登录
- `POST /api/wechat/bind` - 绑定微信账号
- `GET /api/wechat/check-browser` - 检测浏览器环境

**登录流程**：
1. 用户点击"微信登录"
2. 跳转到微信授权页面
3. 用户授权后返回code
4. 后端获取微信用户信息
5. 检查是否已绑定
6. 未绑定 → 显示绑定页面
7. 已绑定 → 直接登录

**绑定流程**：
1. 显示微信头像和昵称
2. 用户输入员工账号和密码
3. 后端验证账号密码
4. 绑定成功 → 登录成功

#### 1.3 鸿蒙系统适配

**功能描述**：
- 自动检测鸿蒙设备
- 鸿蒙设备默认使用账号密码登录
- CSS样式针对高分辨率屏幕优化
- 安全区域适配

**检测规则**：
- User-Agent包含"HarmonyOS"
- User-Agent包含"HuaweiBrowser"
- User-Agent包含"Huawei"

## 核心功能详解

### 微信登录绑定功能

#### 功能概述

微信登录绑定功能是系统的核心特性之一，允许员工使用微信账号快速登录系统，同时确保微信用户与现有员工信息关联。

#### 技术实现

**后端实现**：
- WeChatAuthService：处理微信登录逻辑
- WeChatController：提供微信登录接口
- WeChatConfig：配置微信参数

**前端实现**：
- Login.vue：登录页面，支持绑定UI
- WeChatCallback.vue：微信授权回调页面

#### 安全机制

1. **账号密码验证**：绑定时需要验证员工账号和密码
2. **OpenID唯一性**：每个微信用户的OpenID唯一
3. **防止重复绑定**：一个微信账号只能绑定一个员工
4. **会话管理**：使用JWT Token进行会话管理

#### 用户体验

1. **首次登录引导**：显示绑定页面，引导用户绑定
2. **微信信息展示**：显示微信头像和昵称
3. **自动登录**：绑定成功后，后续登录无需输入密码
4. **取消绑定**：用户可以取消绑定，选择其他登录方式

### 鸿蒙系统适配

#### 功能概述

鸿蒙系统适配功能确保系统在华为鸿蒙设备上正常显示和使用。

#### 技术实现

**浏览器检测**：
- 检测User-Agent中的"HarmonyOS"关键词
- 检测User-Agent中的"HuaweiBrowser"关键词
- 检测User-Agent中的"Huawei"关键词

**样式优化**：
- 针对高分辨率屏幕优化CSS
- 使用安全区域适配（safe-area-inset）
- 优化字体大小和间距

#### 适配效果

- 鸿蒙设备默认使用账号密码登录
- UI元素在鸿蒙设备上显示正常
- 触摸操作流畅
- 字体清晰易读

### 部署步骤

#### 1. 数据库初始化与完整性检查

```bash
# 创建数据库
mysql -u root -p
CREATE DATABASE IF NOT EXISTS canteen_ordering_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;

# 执行数据库初始化脚本
mysql -u root -p canteen_ordering_system < src/main/resources/sql/init.sql

# 运行数据库完整性检查
chmod +x check_database_integrity.sh
./check_database_integrity.sh
```

#### 2. 克隆代码

```bash
mkdir -p /opt/canteen
cd /opt/canteen
git clone --branch V1.2.3 https://github.com/jessonnet/dingcan.git .
```

#### 3. 配置数据库连接

```bash
# 启动MariaDB
systemctl start mariadb
systemctl enable mariadb

# 设置root密码（如果还没有设置）
mysql -uroot -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '123456'; FLUSH PRIVILEGES;"
```

#### 4. 编译后端

```bash
cd /opt/canteen
mvn clean package -DskipTests
```

#### 5. 编译前端

```bash
cd /opt/canteen/frontend
npm install
npm run build
```

#### 6. 部署文件

```bash
mkdir -p /var/www/canteen/backend
mkdir -p /var/www/canteen/frontend
mkdir -p /var/www/canteen/logs

cp target/canteen-ordering-system-*.jar /var/www/canteen/backend/
cp -r frontend/dist/* /var/www/canteen/frontend/
cp src/main/resources/application.yml /var/www/canteen/backend/

chmod -R 755 /var/www/canteen/frontend
chown -R www-data:www-data /var/www/canteen/frontend

chmod -R 755 /var/www/canteen/backend
chown -R root:root /var/www/canteen/backend
```

#### 7. 配置Nginx

```bash
cp nginx/canteen.conf /etc/nginx/sites-available/
ln -sf /etc/nginx/sites-available/canteen.conf /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

#### 8. 配置Systemd服务

```bash
cp nginx/canteen-backend.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable canteen-backend.service
systemctl start canteen-backend.service
```

#### 9. 验证部署

```bash
systemctl status mariadb
systemctl status nginx
systemctl status canteen-backend.service

netstat -tlnp | grep -E '80|8080|3306'

curl http://localhost:8080/api/auth/login -X POST -H "Content-Type: application/json" -d '{"username":"admin","password":"admin123"}'
curl -I http://localhost/
```

```bash
cd /opt/canteen
./check_database_integrity.sh
```
